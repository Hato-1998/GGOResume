FX Preview Browser

사내 에디터 툴
역할
Tool Programmer
회사
콩스튜디오
엔진
Unity Editor
기술
C#, Reflection

개요

FX Preview Browser는 콩스튜디오의 비개발 직군(기획, 아트팀)을 위해 개발한 사내 전용 에디터 툴입니다. 수백 개의 게임 이펙트를 깔끔한 인터페이스에서 빠르게 검색하고 미리보기할 수 있도록 설계되어, 기획과 아트팀의 작업 효율을 대폭 향상시켰습니다.

기존에는 에디터에서 직접 씬을 오염시키면서 각각의 이펙트를 프리뷰해야 했는데, 이 툴을 통해 완전히 독립된 렌더링 환경에서 리얼타임 프리뷰가 가능해졌습니다. 기술적으로는 Unity의 에디터 렌더링 API와 Reflection을 활용한 고급 기법이 적용되었습니다.

핵심 기여

기획/아트 파이프라인 개선을 위한 에디터 툴 개발

씬(Scene) 오염 없이 수백 개의 이펙트를 에디터 창에서 즉시 렌더링하고 확인할 수 있는 사내 전용 FX 뷰어를 개발하여 비개발 직군의 작업 효율을 대폭 향상시킴.

이전에는 이펙트 확인을 위해 매번 씬에 배치해야 했는데, 이 툴 덕분에 별도의 전담 뷰어에서 목록 검색으로 빠르게 비교 검토할 수 있게 되었습니다. 기획과 아트팀의 피드백 사이클이 크게 단축되었습니다.

PreviewRenderUtility 기반 독립 렌더링 환경 구축

에디터 윈도우 내부에 가상 카메라와 조명을 세팅하고 마우스 입력(Pan, Orbit, Zoom) 처리 로직을 구현하여 직관적인 UX를 제공함.

PreviewRenderUtility를 통해 메인 씬과는 완전히 분리된 렌더링 파이프라인을 만들어, 아트팀이 마치 별도의 뷰어에서 작업하는 것처럼 자연스러운 경험을 제공했습니다.

에디터 환경 내 시뮬레이션 수동 동기화

플레이 모드가 아닌 상태에서도 ParticleSystem.Simulate와 CustomAnimator에 강제로 deltaTime을 주입하여 인게임과 동일한 시각적 피드백을 구현함.

에디터의 정지 상태에서도 파티클 시뮬레이션을 수동으로 진행시켜 인게임과 정확히 동일한 비주얼을 미리볼 수 있게 했습니다. 이는 기획/아트의 최종 검증 단계에서 매우 중요한 신뢰성을 제공했습니다.

C# Reflection을 활용한 내부 변수 제어 및 릭 방지

커스텀 애니메이터의 인스턴스 매테리얼 누수 문제를 해결하기 위해, Reflection(BindingFlags.NonPublic)을 사용해 숨겨진 private 필드에 접근, sharedMaterial을 강제 할당하여 메모리 최적화 및 렌더링 오류를 방어함.

매번 이펙트를 렌더링할 때마다 메모리 누수가 발생하는 문제를 Reflection을 활용해 엔진의 내부 구조에 접근하는 방식으로 해결했습니다. 이는 매우 고급 기법이지만, 비용-효율을 생각했을 때 가장 실용적인 솔루션이었습니다.
